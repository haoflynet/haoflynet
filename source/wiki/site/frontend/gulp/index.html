能够自动化处理常见任务： 搭建web服务器 文件保存时自动重载浏览器 使用预处理如Sass、LESS 优化资源，比如压缩CSS、JavaScript、压缩图片

安装npm install gulp -g 初始化项目npm init，会让你输入一些项目的信息，然后创建一个package.json文件

将gulp局部安装到该项目里面去

npm install gulp --save-dev 这样会生成一个node_modules目录，并且会将gulp写入到package.json里面去

gulp目录结构 app # 开发目录，所有的源文件都在这儿 css fonts images index.html js scss dist # 存放生产环境下的内容 gulpfile.js
node_modules/ package.json

然后在根目录创建gulpfile.js来创建gulp任务，内容如下： var gulp = require('gulp'); # 这个告知Node去node_modules中查找包，先局部查找，找不到就去全局查找，找到后赋值给gulp变量

建立一个任务： gulp.task('task_name', funciton(){ 任务内容，比如： return gulp.src('source-files') # 源文件 .pipe(aGulpPlugin()) # 插件处理 .pipe(gulp.dest('destination')) # 输出到目标地址 });

这样在命令行执行gulp task_name就可以运行该任务了

使用sass npm install gulp-sass --save-dev 在gulpfile.js中引入 var sass = require('gulp-sass'); 可以这样写任务了： return gulp.src('source-files').pipe(sass()).pipe(gulp.dest('destination')) 比如： return gulp.src('app/scss/styles/scss').pipe(sass()).pipe(gulp.dest('app/css')) 将sass编译为css 可以使用通配符 return gulp.src('app/scss/*/.scss')....

Gulp提供watch方法自动执行任务 gulp.watch('files-to-watch', ['tasks', 'to', 'run']);比如 gulp.watch('app/scss/*/.scss', ['scss']);

可以将监听变为任务 gulp.task('watch', function(){ gulp.watch('app/scss/*/.scss', ['sass']); }) 这样在终端执行gulp watch就可以监听了，有修改自动就那个了

使用browser sync实现自动刷新 npm install browser-sync --save-dev var browserSync = require('browser-sync');

gulp.task('browserSync', function(){ browserSync({ server: { baseDir: 'app' }, }) }) 修改之前的代码就是： gulp.task('sass', function(){ return gulp.src('app/scss//*.scss') .pipe(sass()) .pipe(gulp.dest('app/css')) .pipe(browserSync.reload({ stream: true }) }); gulp.task('watch', ['browserSync', 'sass'], function (){ gulp.watch('app/scss//.scss', ['sass']); // Other watchers }) gulp.task('watch', ['browserSync', 'sass'], function (){ gulp.watch('app/scss//.scss', ['sass']); // Reloads the browser whenever HTML or JS files change gulp.watch('app/.html', browserSync.reload); gulp.watch('app/js//.js', browserSync.reload); });
